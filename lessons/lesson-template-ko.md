# Stage XX: 단계 제목

## 1. 이 단계에서 배우는 것

이 단계에서 배울 보안 개념을 쉬운 말로 설명합니다. 어려운 용어가 필요하면 처음 나올 때 뜻을 풀어 씁니다.

## 2. 정상 기능 설명

사용자가 원래 하려고 하는 정상 흐름을 설명합니다. 예: 로그인, 프로젝트 조회, 파일 업로드, AI 설명 생성, 관리자 화면 보기.

## 3. 취약한 부분 설명

의도적으로 취약하게 만든 코드나 설정을 설명합니다. 실제 공격 절차가 아니라, 어떤 신뢰 가정이 잘못되었는지에 집중합니다.

## 4. 왜 위험한가

이 문제가 실제 서비스에서 어떤 피해로 이어질 수 있는지 설명합니다. 예: 다른 사용자의 데이터 노출, 저장된 콘텐츠 변조, 권한 없는 관리자 기능 접근.

## 5. 직접 확인하는 방법

로컬에서만 확인할 수 있는 안전한 재현 방법을 씁니다. 외부 사이트, 공개 IP, 실제 서비스 계정은 사용하지 않습니다.

```sh
pnpm scan -- --stage stage-XX-name
```

## 6. BTS_sec로 스캔하는 방법

BTS_sec가 설치되어 있으면 실제 스캔 명령을 사용합니다. 설치되어 있지 않으면 이 저장소의 결정적 scaffold 리포트 생성 명령을 사용합니다.

```sh
make scan STAGE=XX
pnpm compare -- --stage XX
```

## 7. 리포트에서 봐야 할 문장

BTS_sec 리포트가 반드시 설명해야 하는 핵심 문장을 적습니다. 잘못된 설명이 나오면 안 되는 문장도 함께 적습니다.

- 포함되어야 할 말:
  - 핵심 위험
  - 영향을 받는 파일 또는 라우트
  - 안전한 수정 방향
- 나오면 안 되는 말:
  - 이 단계와 다른 취약점 이름

## 8. 수정 방향

기능을 없애지 않고 안전하게 고치는 방향을 설명합니다. 가능하면 취약한 패턴과 안전한 패턴을 나란히 보여줍니다.

## 9. 수정 후 재스캔

수정 후 같은 명령으로 다시 스캔하고 비교합니다. Critical과 High가 사라졌는지, Low/Info가 있다면 실제로 조치할 만한 제안인지 확인합니다.

```sh
make scan STAGE=XX
pnpm compare -- --stage XX
```
