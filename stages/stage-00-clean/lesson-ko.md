# Stage 00: 깨끗한 기준점

이 단계는 BTS_sec의 오탐(false positive)을 확인하기 위한 기준점입니다.

취약점을 일부러 넣지 않은 작은 SaaS 앱을 실행해 보고, 스캐너가 심각한 문제를 잘못 보고하지 않는지 확인합니다.

## 왜 중요한가요?

보안 스캐너는 실제 취약점을 잘 찾아야 하지만, 안전한 코드까지 위험하다고 자주 말하면 사용자가 결과를 믿기 어렵습니다. 그래서 깨끗한 기준점이 필요합니다.

## 이 단계의 안전한 패턴

- `?userId=alice`, `?userId=bob`, `?userId=admin`으로 명시적인 로컬 로그인 시뮬레이션을 사용합니다.
- 프로젝트 상세 조회 전에 서버에서 소유자 확인을 합니다.
- 관리자 페이지는 Admin 데모 사용자만 볼 수 있습니다.
- 공개 포트폴리오 페이지는 공개 프로젝트만 보여줍니다.
- 파일 업로드는 실제 파일을 저장하지 않고, 파일 이름과 MIME 타입과 크기 메타데이터를 검증하는 안전한 stub만 사용합니다.
- AI 텍스트 생성은 외부 API를 호출하지 않는 결정적 로컬 placeholder입니다.
- 실제 시크릿, 실제 OAuth, 실제 결제, 실제 클라우드, 실제 AI API를 사용하지 않습니다.

## 기대 결과

BTS_sec는 Critical과 High 취약점을 0개로 보고해야 합니다. Medium도 가능하면 0개여야 합니다. Low나 Info는 로컬 실습 환경에 대한 설명 수준이라면 허용할 수 있습니다.
